# SmartCalc Mobile

Android-first Flutter-приложение для теплотехнических расчетов ограждающих конструкций. Проект опирается на пользовательские сценарии `smartcalc.ru`, но не копирует сайт, его код или данные.

## Текущее состояние

- Инициализирован Flutter-проект и каркас модульной архитектуры.
- Зафиксированы ТЗ, план реализации, правила разработки, архитектурное решение и чек-лист.
- Добавлены seed-данные по климату, материалам, нормативным ссылкам и сезонным правилам влагорежима.
- Реализован нормативный экран `thermocalc`: расчет сопротивления теплопередаче, температурный профиль по слоям, сезонный расчет влагорежима, график парциального давления против насыщения и ссылки на примененные нормы.
- Добавлены эталонные reference cases и widget/unit-тесты для расчета теплозащиты и влагорежима.
- Подключено локальное `drift`-хранилище проектов с bootstrap demo-проекта и выбором активного проекта на `Dashboard`.
- Реализован рабочий `PDF`-экспорт из `thermocalc`: генерация отчета, локальное сохранение файла и базовый пользовательский flow с индикацией результата.
- Добавлены миграции `drift`-схемы, явная версия датасета в проекте и авто-миграция legacy-проектов при чтении.
- Добавлен рабочий `Phase 2`-конструктор дома: помещения, ограждающие элементы, окна/двери, переиспользуемые конструкции, запуск `Thermocalc` из выбранного элемента и сводка ориентировочных теплопотерь по дому с разделением на непрозрачные участки и проёмы.
- Добавлен первый инкремент отопления: приборы по комнатам, локальный каталог шаблонов, ручная корректировка мощности и баланс покрытия теплопотерь.
- Добавлен отдельный экран `Теплопотери здания v1`: расчет по дому, комнатам и элементам с учетом непрозрачных участков, проёмов и установленной мощности отопления.
- В `building heat loss v1` добавлены базовые вентиляционные потери по комнате.
- В `building heat loss v1` добавлена `инфильтрация v1` по внешним окнам и дверям с preset-герметичности проема.
- В `building heat loss v1` добавлен внутренний теплообмен между смежными комнатами через общую конструкцию внутренней перегородки.
- Добавлен отдельный модуль `Полы по грунту v1`: persisted-расчеты по проекту, поддержанный сценарий `плита по грунту`, проверка итогового `R` и расчет теплопотерь через пол.
- `House plan v2` доведен до `инкремента 2.1`: новое помещение создается через sheet параметров, размещается как draft на плане с проверкой примыкания, а после подтверждения получает room-level наружные стены, пол и верхнее ограждение.

## Быстрый старт

```bash
flutter pub get
flutter run
```

При запуске на `Linux` приложение должно использовать телефонный форм-фактор, а не полноразмерное desktop-окно.

## Замечание по точности

Текущая кодовая база уже закрывает теплозащиту `v1`, сезонный расчет влагорежима, `PDF`-экспорт по конструкции, конструктор дома с помещениями/ограждениями/проёмами, отопительные приборы по комнатам, `building heat loss v1` с базовой вентиляцией, инфильтрацией и межкомнатным теплообменом, `ground floor v2` и `house plan v2 / инкремент 2.1`. При этом внешняя синхронизация и часть следующих инженерных сценариев пока не реализованы, поэтому проект все еще следует трактовать как мобильный инженерный `offline-first` инструмент поэтапного развития, а не как полный аналог настольного расчетного комплекса.
