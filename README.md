# SmartCalc Mobile

Android-first Flutter-приложение для теплотехнических расчетов ограждающих конструкций. Проект опирается на пользовательские сценарии `smartcalc.ru`, но не копирует сайт, его код или данные.

## Текущее состояние

- Инициализирован Flutter-проект и каркас модульной архитектуры.
- Зафиксированы ТЗ, план реализации, правила разработки, архитектурное решение и чек-лист.
- Добавлены seed-данные по климату, материалам, нормативным ссылкам и сезонным правилам влагорежима.
- Реализован нормативный экран `thermocalc`: расчет сопротивления теплопередаче, температурный профиль по слоям, сезонный расчет влагорежима, график парциального давления против насыщения и ссылки на примененные нормы.
- Добавлены эталонные reference cases и widget/unit-тесты для расчета теплозащиты и влагорежима.
- Подключено локальное `drift`-хранилище проектов с bootstrap demo-проекта и выбором активного проекта на `Dashboard`.
- Реализован рабочий `PDF`-экспорт из `thermocalc`: генерация отчета, локальное сохранение файла и базовый пользовательский flow с индикацией результата.
- Добавлены миграции `drift`-схемы, явная версия датасета в проекте и авто-миграция legacy-проектов при чтении.
- Добавлен рабочий `Phase 2`-конструктор дома: помещения, ограждающие элементы, окна/двери, переиспользуемые конструкции, запуск `Thermocalc` из выбранного элемента и сводка ориентировочных теплопотерь по дому с разделением на непрозрачные участки и проёмы.

## Замечание по точности

Текущий экран расчета закрывает теплозащиту `v1`, первый рабочий сезонный расчет влагорежима по локальному dataset и базовый `PDF`-отчет для текущей конструкции. Конструктор дома уже позволяет собрать проект по помещениям, ограждениям и проёмам, но геометрический `canvas`-редактор, отопительные приборы и внешняя синхронизация пока не реализованы, поэтому результат нужно трактовать как инженерный экран предварительной проверки, а не как исчерпывающий нормативный отчет.
