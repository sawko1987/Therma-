# План реализации

## Этап 0. База проекта

- Инициализировать Flutter-репозиторий.
- Создать документацию, `ADR` и чек-лист.
- Зафиксировать доменную структуру и первичную навигацию.

Критерий приемки: проект запускается, документация лежит в репозитории, чек-лист заполнен стартовыми задачами.

## Этап 1. Каталоги и доменная модель

- Ввести модели климата, материалов, ссылок на нормы и проектов.
- Добавить seed-данные в `assets/data`.
- Реализовать загрузку каталогов из локальных `JSON`.

Критерий приемки: приложение отображает статистику по локальным каталогам и пример проекта.

## Этап 2. Preview thermocalc

- Собрать экран `thermocalc` в мобильной компоновке.
- Отрисовать сечение конструкции через `CustomPainter`.
- Отрисовать базовые графики по расчетным сериям.

Критерий приемки: экран показывает пример расчета, слои, графики и статусные индикаторы.

## Этап 3. Нормативный движок

- Реализовать формулы и проверки по выбранным нормам.
- Подготовить набор эталонных кейсов.
- Сверить результаты с публичными примерами `smartcalc.ru`.

Критерий приемки: расчетные значения укладываются в согласованные допуски.

## Этап 4. Сохранение и отчет

- Подключить локальное хранилище проектов.
- Добавить экспорт отчетов.
- Версионировать датасеты и формат проекта.

Критерий приемки: проект открывается повторно, результат можно экспортировать без потери структуры.

## Этап 5. Phase 2

- Схема дома и связь элементов здания с конструкциями.
- Интеграция с `Yandex Disk`.
- Дополнительные калькуляторы.

Критерий приемки: отдельный пересмотр ТЗ после стабилизации `MVP`.
