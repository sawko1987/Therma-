# План реализации

## Шаг 0. Информация об объекте

- Завести объект и выбор климатической точки.
- Подготовить корневую модель проекта под дальнейшую сборку расчета.

Критерий приемки: проект содержит объектные данные и климат, которые могут быть использованы следующими шагами.

## Шаг 1. Создание конструкций

- Ввести модели конструкций и слоев.
- Добавить локальные каталоги материалов и нормативных ссылок.
- Собрать preview одной конструкции с визуализацией слоев и графиков.

Критерий приемки: в проекте есть библиотека конструкций, а preview показывает характеристики выбранной конструкции.

## Шаг 2. Расчетный объект и теплопотери

### Шаг 2.1. Сборка объекта и потери через ограждения

- Собрать `wizard`: объект, комнаты, поверхности и проемы.
- Нормализовать расчетный вход через `BuildingCalculationAssembler`.
- Посчитать потери через ограждения и проемы по комнатам и по объекту.

### Шаг 2.2. Полный баланс

- Добавить вентиляцию/инфильтрацию по комнате.
- Сформировать полный баланс теплопотерь по каждой комнате и по объекту.
- Зафиксировать `heatLossCoefficientWPerK` как публичный контракт для шага 3.

Критерий приемки: пользователь может собрать расчетный объект и увидеть теплопотери по каждой комнате и итог по объекту.

## Шаг 3. Сезонное потребление отопления

- Использовать результат шага 2 для расчета сезонного потребления.
- Показать сценарии отопления на газу, электричестве и тепловом насосе.
- Подготовить точки расширения для тарифов, COP и экономического сравнения.

Критерий приемки: для одного и того же объекта отображается сезонное потребление по выбранным источникам энергии без повторной сборки модели.

## Шаг 4. Сохранение и отчет

- Подключить локальное хранилище проектов.
- Добавить экспорт отчетов.
- Версионировать датасеты и формат проекта.

Критерий приемки: проект открывается повторно, результат можно экспортировать без потери структуры.

## Шаг 5. Phase 2

- Схема дома и связь элементов здания с конструкциями.
- Интеграция с `Yandex Disk`.
- Дополнительные калькуляторы и расширенные режимы расчета.

Критерий приемки: отдельный пересмотр ТЗ после стабилизации `MVP`.
