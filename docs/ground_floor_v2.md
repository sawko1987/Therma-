# Полы по грунту v2

## Что реализовано

- Добавлены поддержанные сценарии `пол по грунту на ленте` и `плита над подвалом`.
- В `GroundFloorCalculation` добавлена nullable-привязка `houseElementId` к floor-элементу из `houseModel`.
- Формат проекта поднят до `v16` с auto-migration legacy-расчетов без link.
- Экран `Полы по грунту` умеет работать в двух режимах: standalone и linked из `house scheme`.
- Для linked flow площадь и конструкция пола синхронизируются от floor-элемента дома.
- `Building heat loss` использует linked-result для floor-элементов вместо unresolved-сценария.

## Что входит в v2

- `Плита по грунту` как baseline-сценарий `v1`.
- `Пол по грунту на ленте` как отдельная поддержанная ветка расчета.
- `Плита над подвалом` как product-flow через `ground floor` с переиспользованием нормативного floor-расчета.
- Открытие `ground floor` прямо из floor-элемента дома для сценариев `onGround` и `overBasement`.
- Создание связанного расчета по выбранному floor-элементу с prefill от комнаты и floor-конструкции.
- Автосинхронизация `constructionId` и `areaSquareMeters` у linked-расчета при изменении геометрии комнаты и room-level пола.

## Что пока не входит

- `Пол над техподпольем` и `пол над проездом` через модуль `ground floor`.
- Автосинхронизация периметра и геометрических параметров linked-расчета по сложному multi-cell контуру комнаты.
- Отдельный `PDF`-отчет по модулю `ground floor`.

## Принятая модель v2

- `slabOnGround` и `stripFoundationFloor` требуют floor-конструкцию типа `Пол по грунту`.
- `basementSlab` требует floor-конструкцию типа `Пол над подвалом`.
- Для `basementSlab` добавочное сопротивление грунта не применяется; используется нормативный floor-расчет перекрытия над неотапливаемым подвалом.
- Для house-level summary `onGround` без linked-расчета остается unresolved-сценарием.
- Для `overBasement` linked-расчет имеет приоритет, но legacy direct fallback в `thermocalc` сохранен для обратной совместимости.
