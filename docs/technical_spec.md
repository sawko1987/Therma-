# Техническое задание

## Цель

Сделать мобильное Flutter-приложение, которое покрывает ключевой сценарий `smartcalc.ru/thermocalc`: подбор и оценка ограждающей конструкции по теплотехнике, влагонакоплению и теплопотерям.

## Обязательный объем MVP

- Платформа: `Android first`.
- Режим работы: `offline-first`, без обязательного сервера и без авторизации.
- Основной модуль: один калькулятор ограждающих конструкций.
- Визуализация: сечение конструкции и инженерные графики на экране результата.
- Отчетность: экранный отчет, подготовленная архитектура под `PDF`.
- Данные: локальные справочники климатических точек, материалов и нормативных ссылок.

## Основные сценарии

- Пользователь открывает проект и выбирает климатическую точку.
- Пользователь задает тип помещения и тип конструкции.
- Пользователь формирует набор слоев конструкции.
- Приложение отображает расчетные показатели, графики и визуальное сечение.
- Пользователь сохраняет проект локально и позже возвращается к нему.
- На шаге 2 пользователь добавляет помещение сразу на план, задает длину, ширину и высоту, а затем перемещает помещение по схеме без пересечения с другими помещениями.

## Ограничения

- Не копировать код, HTML, стили, изображения и внутренние таблицы исходного сайта.
- Не встраивать полный текст нормативных документов в приложение.
- Использовать только краткие ссылки на документы, пункты и происхождение данных.
- Доменная модель должна быть готова к `Phase 2`: схеме дома, `Yandex Disk`, дополнительным калькуляторам.

## Что остается вне MVP

- Интерактивная схема дома с выбором конструкции на схеме.
- Синхронизация или резервное копирование через `Yandex Disk`.
- Следующие специализированные калькуляторы поверх базового `MVP`, включая окна и расширенные сценарии полов по грунту.
- Полноценная облачная синхронизация и аккаунты.
