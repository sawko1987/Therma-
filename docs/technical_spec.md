# Техническое задание

## Цель

Сделать мобильное Flutter-приложение, которое покрывает пошаговый сценарий расчета частного дома или другого небольшого объекта: данные об объекте, библиотека конструкций, сборка расчетного объекта, теплопотери по комнатам и объекту, а затем сезонное потребление отопления по типу источника энергии.

## Обязательный объем MVP

- Платформа: `Android first`.
- Режим работы: `offline-first`, без обязательного сервера и без авторизации.
- Основной сценарий: `шаг 0 -> шаг 1 -> шаг 2 -> шаг 3`.
- `Шаг 0`: сведения об объекте и выбор климата.
- `Шаг 1`: создание и хранение библиотеки конструкций.
- `Шаг 2`: сборка расчетного объекта из комнат, поверхностей и проемов; расчет теплопотерь.
- `Шаг 3`: сезонное потребление отопления для газа, электричества и теплового насоса на основе результата шага 2.
- Визуализация: отдельный экран preview одной конструкции и экран результата по объекту.
- Отчетность: экранный отчет, подготовленная архитектура под `PDF`.
- Данные: локальные справочники климатических точек, материалов и нормативных ссылок.

## Основные сценарии

- Пользователь открывает объект и выбирает климатическую точку.
- Пользователь создает или редактирует библиотеку конструкций.
- Пользователь заводит комнаты объекта и задает их параметры.
- Пользователь назначает комнатам ограждающие поверхности, типы границ и проемы.
- Приложение считает теплопотери через ограждения, вентиляцию и полный баланс по каждой комнате и по объекту.
- Пользователь переходит к шагу 3, где на основе коэффициента теплопотерь и итогового баланса рассчитывается сезонное потребление отопления.

## Ограничения

- Не копировать код, HTML, стили, изображения и внутренние таблицы исходного сайта.
- Не встраивать полный текст нормативных документов в приложение.
- Использовать только краткие ссылки на документы, пункты и происхождение данных.
- Шаг 2 обязан выдавать стабильный результат для шага 3: итог по объекту, итог по комнатам и `heatLossCoefficientWPerK`.
- Доменная модель должна быть готова к дальнейшему расширению: схема дома, `Yandex Disk`, дополнительные калькуляторы и более детальные сезонные модели.

## Что остается вне текущего MVP

- Интерактивная схема дома с размещением конструкций на плане.
- Синхронизация или резервное копирование через `Yandex Disk`.
- Полноценная облачная синхронизация и аккаунты.
- `PDF`-экспорт и постоянное хранилище проекта.
- Расширенные поправки по мостикам холода, тарифам и стоимости эксплуатации.
