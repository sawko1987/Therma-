# ADR 001: стартовая архитектура SmartCalc Mobile

## Статус

Принято.

## Контекст

Нужно быстро поднять работоспособный Flutter-каркас под инженерный калькулятор, не потеряв контроль над доменной моделью, источниками данных и будущими расширениями.

## Решение

- Использовать `feature-first` структуру модулей.
- Для управления состоянием использовать `flutter_riverpod`.
- Каталоги на старте грузить из локальных `JSON`-assets.
- Репозитории и расчетный движок скрывать за интерфейсами.
- Для визуализации графиков и сечения использовать `CustomPainter`.
- Полноценное локальное хранилище реализовать отдельным этапом после стабилизации доменной модели.

## Последствия

- Приложение уже сейчас запускается и показывает реальные доменные сущности без жесткой связки с конечной БД.
- Переход на `drift` можно выполнить без слома UI, потому что интерфейсы уже выделены.
- Расчетный движок можно эволюционно развивать: текущая реализация уже закрывает теплозащиту `v1`, а влагорежим и дополнительные проверки можно добавлять без слома экрана и провайдеров.

## Примечание

Проверка `pub.dev` на `27 марта 2026` показала, что `sqlite3_flutter_libs` помечен как `EOL`, поэтому выбор конкретной связки для `drift` фиксируется отдельным шагом при реализации хранения.
