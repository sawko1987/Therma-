# План Дома v2

## Цель

Переделать шаг 2 так, чтобы пользователь сначала создавал помещение сразу на плане, затем задавал размеры и двигал его по схеме, а система сама определяла наружные ограждения и смежные комнаты.

## Инкремент 1. Новый flow помещения

- Новое помещение создается через sheet параметров, затем размещается на плане как draft прямоугольник на сетке `0.5 м`.
- Базовый размер нового помещения: `3 x 3 x 2.7 м`.
- После подтверждения размещения помещение сохраняется в проект, а ниже плана остается редактор параметров: название, тип помещения, длина, ширина и высота.
- Помещение нельзя разместить поверх другого ни при создании, ни при перемещении, ни при изменении размеров.
- Если помещение не примыкает к уже собранному дому по общей стороне, сохранить его нельзя.

Статус: выполнен.

## Инкремент 2. Составные помещения и составные стены

- Помещение перестает трактоваться только как один прямоугольник.
- Пользователь может собрать фигурное помещение из нескольких соседних ячеек и удалить перегородку между ними.
- Одна сторона помещения может состоять из нескольких конструктивных участков; для этого наружный контур хранится как набор участков, а не как одна “стена комнаты”.
- Проемы должны жить на конкретных наружных участках.

Статус: выполнен.

Что уже реализовано:

- В модель `Room` добавлены `cells` и расчет габарита через bounding box.
- На плане можно выбрать перегородку между смежными комнатами и удалить ее, объединив комнаты в одно составное помещение.
- Наружную стену можно разрезать на несколько сегментов прямо с плана.
- Для выбранной комнаты добавлен прямой режим ячеек: соседние ячейки `0.5 м` можно добавлять и убирать без обходного flow через временную комнату.
- Пересборка наружных стен сохраняет стабильные auto-wall сегменты, их `constructionId` и привязку проемов там, где у сегмента остается однозначный наследник.
- Single-rect и multi-cell сценарии закреплены widget/unit-тестами.
- Формат проекта для этого инкремента не менялся: `Room.cells` остается persisted-моделью геометрии, а новая миграция не потребовалась.

## Инкремент 3. Перетекание тепла между комнатами

- Для смежных комнат рассчитывается теплообмен через внутренние перегородки.
- Поток между комнатами должен учитываться в комнатном балансе, но не должен искажать общий итог по дому.
- Для расчета используется площадь общей грани и сопротивление внутренней перегородки.

Статус: выполнен.

## Инкремент 2.1. Размещение И Автоограждения Комнаты

- После подтверждения размещения общей стороной новая комната автоматически исключает внутреннюю стену из внешнего контура.
- Для новой комнаты сразу открывается sheet выбора конструкций: внешние wall-сегменты, `Пол` и единый слот `Верх`.
- Для `Верха` выбирается либо `перекрытие`, либо `кровля`; двойной учет верхнего ограждения для одной комнаты не допускается.
- `Пол` и `Верх` синхронизируют площадь от площади комнаты и обновляются при изменении геометрии.
- План дома валидируется как одна связная фигура из примыкающих комнат.

Статус: выполнен.

## Инкремент 4. Диагонали и радиусы без режима AutoCAD

- Диагональные и радиусные стены не рисуются свободно, а задаются через ограниченные шаблоны и приближенные параметры.
- Геометрия должна быть удобной для мобильного интерфейса и не превращать экран в CAD-редактор.
- Кривые участки считаются приближенно, с инженерным, а не чертежным уровнем точности.

Статус: запланировано.

## Ограничения текущего baseline

- Диагональные и радиусные стены пока не поддерживаются.
- Для multi-cell помещений поддерживается редактирование формы через grid-ячейки, а не свободное перетаскивание всего сложного контура как одного объекта.
